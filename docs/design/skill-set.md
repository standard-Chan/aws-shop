# 기술 스택

## 1. 상품 및 리뷰 데이터 저장용 DB: MySQL

선택 이유:

- 사용하는 AWS 데이터가 RDB에 잘 맞는 구조
- 쓰기보다 조회의 비중이 높고, 다양한 조건으로 필터링이 가능해야한다.
- PostgreSQL도 후보였지만, 현재 학습 수준에서 MySQL이 적합하다고 판단하여, MySQL을 선택한다.

---

## 2. Java: 21

선택 이유: 17이 아닌, 21로 진행한 이유

- Spring 환경에서 virtual thread를 활용하면, I/O 대기가 많은 구조에서 처리량 개선 가능성이 있다.
- DB 조회, 검색 요청, 외부 I/O가 많은 서비스 특성에 적합하다.

---

## 3. 아키텍처: DDD

DDD를 선택한 이유:

- 대용량 데이터와 기능 확장이 동시에 일어날 때 도메인 경계를 분명히 하기 쉽다.
- 상품, 리뷰, 검색, 사용자, 인덱싱 같은 관심사를 분리할 수 있다.
- 향후 MSA 전환 시 서비스 경계를 자연스럽게 가져갈 수 있다.
