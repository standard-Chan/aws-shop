# Analytics 3번째 구현 계획

## Summary

3번째 구현은 `analytics_events` 테이블을 기반으로 Funnel 분석 조회 API를 추가한다.

- 기준은 이벤트 수 기반 퍼널이다.
- 기간 조건은 `from`, `to`를 필수로 받는다.
- 기간은 `occurred_at >= from AND occurred_at < to`로 조회한다.
- KPI, Superset, 사용자 수 기반 퍼널, 세그먼트, LLM 인사이트는 이번 범위에서 제외한다.

## API Contract

- Method: `GET`
- URL: `/api/analytics/funnel`
- Query Parameters:
  - `from`: 필수, ISO-8601 Instant 문자열
  - `to`: 필수, ISO-8601 Instant 문자열

예시:

```http
GET /api/analytics/funnel?from=2026-05-01T00:00:00Z&to=2026-06-01T00:00:00Z
```

응답:

```json
{
  "from": "2026-05-01T00:00:00Z",
  "to": "2026-06-01T00:00:00Z",
  "basis": "EVENT_COUNT",
  "steps": [
    {
      "eventType": "SEARCH",
      "count": 10000,
      "conversionRateFromPrevious": null,
      "conversionRateFromSearch": 1.0
    },
    {
      "eventType": "PRODUCT_VIEW",
      "count": 4000,
      "conversionRateFromPrevious": 0.4,
      "conversionRateFromSearch": 0.4
    },
    {
      "eventType": "ADD_TO_CART",
      "count": 1200,
      "conversionRateFromPrevious": 0.3,
      "conversionRateFromSearch": 0.12
    },
    {
      "eventType": "PURCHASE",
      "count": 300,
      "conversionRateFromPrevious": 0.25,
      "conversionRateFromSearch": 0.03
    }
  ]
}
```

## Implementation Changes

- `AnalyticsEventRepository`에 기간 내 `eventType`별 count 집계 쿼리를 추가한다.
- `AnalyticsFunnelService`가 고정된 퍼널 순서로 count와 전환율을 계산한다.
- 고정 순서:
  - `SEARCH`
  - `PRODUCT_VIEW`
  - `ADD_TO_CART`
  - `PURCHASE`
- 조회 결과가 없는 단계는 count `0`으로 채운다.
- 전환율 규칙:
  - `conversionRateFromPrevious = 현재 단계 count / 직전 단계 count`
  - `conversionRateFromSearch = 현재 단계 count / SEARCH count`
  - 분모가 `0`이면 `0.0`
  - `SEARCH.conversionRateFromPrevious`는 `null`
  - `SEARCH.conversionRateFromSearch`는 검색 count가 `0`이면 `0.0`, 아니면 `1.0`
- `from >= to`이면 `400 Bad Request`를 반환한다.

## Test Plan

- Repository 통합 테스트:
  - 기간 내 이벤트만 event type별 count에 포함되는지 검증한다.
  - `to` 경계값은 제외되는지 검증한다.
- Service 단위 테스트:
  - 단계 순서가 고정되는지 검증한다.
  - 전환율 계산을 검증한다.
  - 분모가 0일 때 `0.0`으로 처리되는지 검증한다.
- Controller 테스트:
  - 정상 요청 시 `200 OK`와 funnel 응답을 반환하는지 검증한다.
  - `from`, `to` 누락 또는 잘못된 형식 요청은 `400 Bad Request`인지 검증한다.
  - `from >= to` 요청은 `400 Bad Request`인지 검증한다.
- 최종 검증:
  - `./gradlew test --tests '*Analytics*' --no-daemon -Dorg.gradle.cache.internal.locklistener=false`
  - `./gradlew test --no-daemon -Dorg.gradle.cache.internal.locklistener=false`

## Assumptions

- V1 Funnel은 이벤트 수 기반이며 사용자 수 기반 deduplication은 하지 않는다.
- 기간 기준은 `occurred_at`이며 `created_at`은 조회 조건에 쓰지 않는다.
- 모든 시간 입력은 UTC Instant 문자열을 권장한다.
