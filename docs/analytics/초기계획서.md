다른 세션에 그대로 붙여넣어도 될 정도로 정리해볼게.

# 프로젝트 배경

현재 진행 중인 프로젝트는 **이커머스 서비스**이다.

원래는 일반적인 쇼핑몰의 단순 기능 구현을 넘어 **사용자 행동 데이터 수집 및 분석 플랫폼**으로 확장하려고 한다.

새로운 광고 플랫폼 토이 프로젝트를 만드는 것이 아니라, 기존 이커머스 서비스에 실제 서비스에서 활용 가능한 데이터 파이프라인과 분석 기능을 추가하는 방향으로 진행한다.

목표는:

> 단순 CRUD 개발자가 아니라 사용자 행동 데이터를 수집·분석·활용하는 데이터 기반 백엔드 개발자임을 보여주는 것

이다.

---

# 왜 이 프로젝트를 하는가

인크로스 채용공고 분석 결과, 실제로 요구하는 역량은 RTB 엔진(DSP/SSP) 개발보다 아래 영역에 더 가깝다고 판단했다.

* Node.js 기반 API 서버 개발
* Kafka 기반 데이터 파이프라인 구축
* 대용량 데이터 처리 및 분석
* BI 시스템 구축 (Superset)
* 광고 성과 분석
* LLM 기반 서비스 개발

따라서 RTB 엔진을 구현하는 것보다

> "사용자 행동 데이터를 이벤트 단위로 수집하고 분석하여 인사이트를 제공하는 플랫폼"

을 구현하는 것이 채용 포지션과 훨씬 잘 맞는다.

---

# 최종 목표

이커머스 서비스에 아래 기능을 추가한다.

```text
사용자 행동 이벤트 수집
        ↓
Kafka 전송
        ↓
Consumer 처리
        ↓
Analytics DB 적재
        ↓
퍼널 분석
        ↓
BI 대시보드
        ↓
LLM 기반 자동 인사이트 생성
```

---

# 구현 목표

## 1. 사용자 행동 이벤트 수집

사용자의 주요 행동을 이벤트로 정의한다.

### Search Event

사용자가 상품 검색

```json
{
  "eventType": "search",
  "userId": 1,
  "keyword": "맥북",
  "timestamp": "2026-05-29T12:00:00"
}
```

---

### Product View Event

상품 상세 조회

```json
{
  "eventType": "product_view",
  "userId": 1,
  "productId": 100
}
```

---

### Add To Cart Event

장바구니 추가

```json
{
  "eventType": "add_to_cart",
  "userId": 1,
  "productId": 100
}
```

---

### Purchase Event

상품 구매

```json
{
  "eventType": "purchase",
  "userId": 1,
  "orderId": 500
}
```

---

# 2. Kafka 이벤트 파이프라인 구축

이벤트를 Kafka Topic으로 발행한다.

예시:

```text
search-events
product-view-events
cart-events
purchase-events
```

구조:

```text
Frontend
    ↓
Backend API
    ↓
Kafka Producer
    ↓
Kafka Topic
    ↓
Kafka Consumer
```

---

# 3. Analytics DB 구축

서비스 DB와 분석 DB를 분리한다.

예시:

```text
Service DB
    ↓
Kafka
    ↓
Analytics Consumer
    ↓
Analytics DB
```

목적:

* 운영 트랜잭션과 분석 쿼리 분리
* 대용량 집계 가능 구조 확보
* 실무 데이터 플랫폼 구조 학습

---

# 4. Funnel 분석

사용자 행동을 구매 퍼널로 분석한다.

```text
Search
 ↓
Product View
 ↓
Add To Cart
 ↓
Purchase
```

예시:

```text
Search      10000
View         4000
Cart         1200
Purchase      300
```

분석 목표:

* 어느 단계에서 이탈이 발생하는지 확인
* 구매 전환율 계산
* 상품 및 검색어 성과 분석

---

# 5. KPI 계산

광고 플랫폼과 유사한 방식으로 KPI를 계산한다.

## Search CTR

검색 후 상품 클릭률

CTR = Product View / Search

---

## Cart Rate

장바구니 진입률

Cart Rate = Add To Cart / Product View

---

## Purchase Rate

구매 전환율

Purchase Rate = Purchase / Product View

---

# 6. 사용자 관심사 세그먼트 생성

광고 플랫폼의 Audience Segmentation 개념을 적용한다.

예시:

```text
맥북
아이폰
에어팟
```

↓

```text
APPLE_SEGMENT
```

---

```text
축구화
유니폼
풋살공
```

↓

```text
SPORTS_SEGMENT
```

---

예상 테이블:

```sql
user_interest_segment

user_id
segment_name
score
updated_at
```

목적:

* 사용자 행동 기반 프로파일링
* 광고 플랫폼 Audience 개념 학습
* 향후 추천 시스템 활용

---

# 7. BI 대시보드 구축

Apache Superset 사용 예정.

분석 항목:

* 인기 검색어 TOP N
* 조회수 TOP 상품
* 전환율 TOP 상품
* 카테고리별 구매율
* 시간대별 구매율
* 사용자 세그먼트 분포

목적:

* 실무 BI 경험 확보
* 데이터 시각화 경험 확보

---

# 8. LLM 기반 자동 인사이트 생성

단순 챗봇이 목적이 아니다.

운영자가 데이터를 이해하도록 돕는 것이 목적이다.

입력:

```text
검색량 +15%
조회율 -8%
구매율 -12%
```

출력:

```text
검색량은 증가했으나 구매율이 감소했습니다.

특정 상품군의 가격 경쟁력이 낮아졌거나,
상품 상세 페이지 이탈이 증가한 것으로 보입니다.

해당 카테고리 분석을 권장합니다.
```

목표:

* 광고 운영 자동화 개념 경험
* AI 서비스 경험
* 인크로스 공고의 LLM 활용 역량 어필
